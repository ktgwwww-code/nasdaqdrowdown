# NASDAQ100 ドローダウン監視ツール 仕様書

## 1. 目的

NASDAQ100指数（^NDX）の「観測史上最高値からの下落率」を毎日自動で監視し、
あらかじめ定めた閾値（−25% / −30% / −40% / −50%）に到達したときだけ、
Discord にプッシュ通知を送る。

目的は **「相場を毎日見ない（=気絶する）まま、暴落だけは絶対に見逃さない」** こと。
通知が来たときだけ、ポートフォリオのレバ比率を組み替える。

普段は何も通知しない。暴落だけがユーザーを起こす。

---

## 2. 全体構成

- **言語**: Python 3.11+
- **データ取得**: yfinance（`^NDX` = NASDAQ100指数）
- **実行基盤**: GitHub Actions（cron で1日1回・無料）
- **状態保存**: リポジトリ内の JSON ファイル（`state.json`）を読み書きしてコミット
- **通知**: Discord Webhook（URLにPOSTするだけ）
- **通知受信**: スマホの Discord アプリでプッシュ通知

---

## 3. 監視ロジック

### 3.1 高値の基準
- **観測史上最高値（all-time high）からの下落率**で判定する。
- yfinance で `^NDX` を **`period="max"`（全期間）** で取得し、その期間の終値の最高値を「史上最高値」とする。
- 取得後、`state.json` に保存している過去の最高値（`all_time_high`）と比較し、
  新しい最高値が出ていれば更新する。

### 3.2 下落率の計算
```
drawdown = (all_time_high - current_price) / all_time_high * 100
```
- `current_price` = 最新の終値
- `all_time_high` = state.json に記録された史上最高値（当日値で更新後）

### 3.3 閾値とアクション

| 閾値（史上最高値比） | アクション（通知文に記載） |
|---|---|
| **−25%** | レバを増額。通常フェーズ「等倍30・レバ5」→「等倍30・レバ10」（L≒1.25） |
| **−30%** | レバをさらに増額。「等倍30・レバ15」（L≒1.33） |
| **−40%** | レバ大幅増額。「等倍30・レバ25」（L≒1.45）。**妻のオルカン資金をレバナスへ寄せる検討開始** |
| **−50%** | レバ最大。「等倍30・レバ30」（L≒1.50）。**妻資金フル投入。妻の同意を取る** |

※ いずれも「等倍は維持してレバを上乗せ」する方式（等倍を削ってレバに替えない）。
※ 増額分の原資は「妻のオルカン資金をレバナス寄りに振り替える」。
※ 判定は必ず **原指数 ^NDX** で行う（レバナスの基準価額では判定しない）。
※ Lはいずれも 1.5 以下（ケリー基準・税引き後の頑健ラインの範囲内）。

### 3.4 再発火防止（状態管理）★重要
- 一度ある閾値で通知したら、**史上最高値が更新される（=新高値を付ける）まで、同じ閾値では再通知しない**。
- これがないと、暴落中ずっと毎日通知が飛んで「気絶」が壊れる。
- 実装：state.json に「現在どの閾値まで発火済みか」を持つ。
  - 例：`"fired_level": 30` なら −30% まで通知済み。
  - さらに下落して −40% に達したら −40% を通知し `fired_level` を 40 に更新。
  - 史上最高値が更新されたら `fired_level` を 0 にリセット（再装填）。

---

## 4. state.json の構造（例）

```json
{
  "all_time_high": 22000.0,
  "fired_level": 0,
  "last_run": "2026-06-16",
  "last_price": 21500.0
}
```

- `all_time_high`: これまでに観測した史上最高値（終値ベース）
- `fired_level`: 直近で通知済みの閾値（0/25/30/40/50）。0 = 未発火
- `last_run`: 最終実行日
- `last_price`: 最終取得価格（デバッグ・ログ用）

---

## 5. 処理フロー（擬似コード）

```
1. state.json を読み込む（なければ初期化：all_time_high=0, fired_level=0）
2. yfinance で ^NDX を period="max" で取得し、終値系列を得る
3. 系列の最高値 hist_high と、最新終値 current_price を取得
4. all_time_high = max(state.all_time_high, hist_high, current_price)
5. もし all_time_high が前回より更新されていたら:
      fired_level = 0   # 新高値で再装填（リセット）
6. drawdown = (all_time_high - current_price) / all_time_high * 100
7. 到達している最大の閾値 level を判定（drawdown >= 50 → 50, >=40 → 40, >=30 → 30, >=25 → 25, else 0）
8. もし level > fired_level なら:
      Discord に通知を送る（下落率・到達閾値・アクション文を含む）
      fired_level = level
9. state.json を更新して保存・コミット
```

※ 7-8 の比較を `level > fired_level` にすることで、
   「−25%通知済みのあと、さらに−30%に達したら−30%だけ追加通知」が自然に実現する。
   同じ−25%圏内に留まる限り再通知しない。

## 6. Discord 通知の中身

### 6.1 送信方法
- Discord サーバーのチャンネルで Webhook URL を発行。
- その URL に JSON を POST するだけ（`requests.post`）。
- Webhook URL は **GitHub Secrets** に格納（`DISCORD_WEBHOOK_URL`）。コードに直書きしない。

### 6.2 通知メッセージ例（−30%到達時）

```
🔴 NASDAQ100 ドローダウン警報

史上最高値: 22,000
現在値: 15,400
下落率: −30.0%

▶ アクション: レバをさらに増額
　通常フェーズ「等倍30・レバ15」へ組み替え（L≒1.33）
　原資: 妻のオルカン資金をレバナスへ振り替え

※ 判定は原指数 ^NDX。レバナス基準価額ではない。
※ 売らない。安く拾えるボーナス。握る論理を思い出すこと。
```

- 閾値ごとに「アクション文」を出し分ける（3.3 の表に対応）。
- 文末に毎回「**売らない。握る論理を思い出すこと。**」を入れる
  （暴落の渦中で恐怖に飲まれないための、自分への補給線）。

## 7. GitHub Actions 設定

### 7.1 実行タイミング
- 1日1回、**米国市場クローズ後**に実行。
- 米国東部の引け（16:00 ET）後、確定した終値を取りたい。
- 日本時間の朝に走らせる。cron は UTC 指定。
  - 例：UTC 22:00（夏時間の米国引け後あたり）。
  - 夏時間/冬時間で米国引けのUTCがずれるので、**余裕を持って UTC 23:00〜翌1:00 あたり**で1回。
  - 推奨 cron 例：`0 0 * * 2-6`（UTC 0:00 = 日本 9:00、米国前営業日の引け後。曜日は実行日基準で火〜土）
  - ※ 厳密な時刻はテストして調整。終値が取れていればOK。

### 7.2 ワークフローでやること
1. リポジトリを checkout
2. Python セットアップ、`pip install yfinance requests`
3. 監視スクリプト実行（Secrets から Webhook URL を渡す）
4. state.json が更新されていたら、コミット＆プッシュ（bot として）

### 7.3 必要な Secrets
- `DISCORD_WEBHOOK_URL`: Discord の Webhook URL

## 8. エラーハンドリング・堅牢性

- **データ取得失敗**（yfinance が空・通信エラー）時は、通知を送らず正常終了（後日リトライ）。
  - ただし「N日連続で取得失敗」したら、その旨だけ Discord に通知（監視が死んでることに気づけるように）。
- **祝日・休場日**：データが前営業日と同じなら、新規判定せずスキップしてよい。
- **state.json が壊れている/無い**：初期値で再生成（all_time_high=0 から始まり、初回取得で埋まる）。
- **二重通知防止**：fired_level による状態管理（3.4）で担保。

## 10. このツールの設計思想（忘れないために）

- **普段は沈黙、暴落だけが起こす。** 毎日チャートを見る生活を作らないことが最大の目的。
  ウィジェットでの常時監視は「見たくなる→触りたくなる」衝動の温床なので採用しない。
- **判定は原指数、行動はレバ比率の組み替えのみ。** 売却・損切りの指示は一切出さない。
- **通知文に「やること」と「握る論理」を必ず載せる。** 暴落の渦中で判断・我慢をしないで済むように、
  冷静な今の自分が、未来の自分へ指示と励ましを先に書いておく。
- **L（実効レバレッジ）は常に 1.5 以下。** どの閾値でもケリー基準の頑健域を超えない。
